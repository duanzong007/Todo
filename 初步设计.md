# 一、项目目标

开发一个 **极简跨平台任务提醒系统**，核心目标：

1. **跨平台**
   - 基于 Web
   - 任意设备浏览器可访问
   - 不依赖 Apple / Android 生态

2. **输入极快**
   - 一个输入框即可添加任务
   - 支持文本输入、粘贴解析、ICS导入

3. **界面极简**
   - 不使用月历视图
   - 不展示未来很多天
   - 只显示当前相关任务

4. **任务持续提醒**
   - 对重要任务持续提醒
   - 对日程类任务自动消失

系统定位：

> 一个“任务捕获 + 持续提醒”的极简系统。

# 三、任务类型（核心模型）

系统只存在 **三种任务类型**。

这是整个系统的核心逻辑。

---

# 1 DDL任务（Deadline）

## 定义

有截止日期的任务。

## 特征

- 有一个截止日期
- 在截止日期之前 **每天都会显示**
- 不会自动消失
- 必须手动确认完成

## 示例

```
周五交作业
3月20号提交论文
```

## 显示效果

```
交作业
DDL还有3天
```

如果已过期：

```
⚠ 已过期
```

## 生命周期

```
创建
↓
每天显示
↓
截止日期
↓
仍然显示
↓
用户确认完成
↓
消失
```

---

# 2 日程任务（Schedule）

## 定义

某一天发生的事件。

## 特征

- 只属于某一天
- 到了当天才会出现
- 当天结束自动消失
- 不需要确认完成

## 示例

```
明天上课
今天开会
下午签到
```

## 生命周期

```
创建
↓
对应日期显示
↓
当天结束
↓
自动消失
```

## 典型使用场景

- 上课
- 会议
- 临时安排
- 课程表

---

# 3 固化Todo（Persistent Todo）

## 定义

没有日期限制的长期待办。

## 特征

- 没有日期
- 一直存在
- 直到手动完成

## 示例

```
买电池
取快递
整理电脑
研究一个项目
```

## 生命周期

```
创建
↓
一直显示
↓
用户完成
↓
消失
```

# 四、输入系统

系统核心是 **一个统一输入框**。

任何输入都会被解析为任务。

---

# 输入来源

系统支持三种输入来源：

### 1 文本输入

用户直接输入：

```
买电池
周五交作业
明天上课
```

系统自动解析任务类型。

---

### 2 粘贴解析（短信）

用户粘贴短信内容。

例如：

```
【菜鸟驿站】取件码 384923
```

系统解析为：

```
取快递
取件码 384923
```

任务类型：

```
Todo
```

任务会一直存在直到确认完成。

---

### 3 ICS导入

用户可以导入 **ICS文件**。

例如：

- 课程表
- 会议安排

系统解析：

```
课程名称
时间
周期
```

导入后生成：

```
Schedule任务
```

例如：

```
周一 高数
周三 高数
```

---

# 五、任务解析规则

系统根据输入内容自动判断任务类型。

---

普通文本：

```
买电池
```

解析为：

```
Todo
```

---

带日期：

```
明天上课
```

解析为：

```
Schedule
```

---

带截止语义：

```
周五交作业
```

解析为：

```
DDL
```

---

短信内容：

```
取件码
```

解析为：

```
Todo
```

---

ICS导入：

```
课程事件
```

解析为：

```
Schedule
```

# 六、界面结构

界面保持极简，不做日历。

页面只显示 **三个区域**：

```
今天
DDL
Todo
```

---

## 示例界面

```
今天

高数课
开会


DDL

交作业 还有2天


Todo

买电池
取快递
```

---

## 界面原则

- 不显示远期日历
- 不显示大量未来任务
- 保持界面简单

---

# 七、任务操作

系统支持基础操作。

---

## 完成任务

用户可以确认完成任务。

适用于：

```
DDL
Todo
```

---

## 延期任务

用户可以延期任务。

适用于：

```
DDL
Schedule
```

示例：

```
延期到明天
延期三天
延期下周
```
