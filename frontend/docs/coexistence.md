# 新旧前端并存策略

## 原则

迁移期间，现有 Go 模板页面继续作为生产路径。

Vue 前端先作为独立工程存在，只负责逐步接管特定页面或页面片段。

## 当前并存方式

- Go 模板继续服务：
  - `/classic`
  - `/me/classic`
  - `/sms/native/classic`
  - `/login`
  - `/register`
- Vue 构建产物由 Go 服务在 `/` 提供
- Vue 构建产物由 Go 服务在 `/me` 提供
- Vue 构建产物由 Go 服务在 `/sms/native` 提供
- Vue + Vite 独立运行在：
  - `http://127.0.0.1:5173`
- Vite 通过代理访问 Go 后端：
  - `/dashboard`
  - `/events`
  - `/tasks`
  - `/imports`
  - `/sms`
  - `/me`

## 迁移方式

### 阶段 1

只验证工程和接口，不接管任何页面。

### 阶段 2

优先迁移 `/me` 管理页。

当前方式：

- Vue 独立工程实现管理页
- Go 新增 `/me/data` JSON 数据端点
- Go 模板旧管理页保留为 `/me/classic`
- Go 服务 `/me` 已切换到 Vue 构建产物
- 批量操作继续复用 `/me/tasks/apply`

### 阶段 3

迁移短信导入页。

注意：

- 保留安卓壳原生桥
- Web 端和壳端继续使用同一套页面
- 旧模板回退路径保留为 `/sms/native/classic`
- 本地历史缓存 key 保持不变

### 阶段 4

迁移首页任务区。

注意：

- 首页是高频核心页，已经切换到 Vue 构建产物
- Go 旧模板首页保留在 `/classic`
- 任务列表、已完成列表、金句空状态、更多面板、添加面板、延期选择器和时间选择器都由 Vue 接管
- 任务业务规则仍然走 Go 后端接口

## 不变量

迁移期间必须保持：

- 现有 URL 可访问
- 现有 session cookie 继续可用
- PWA 与安卓壳不被破坏
- 后端业务语义不重复实现

## 回退策略

每个迁移阶段都要保留旧页面路径，直到新页面稳定。

如果 Vue 页面出现问题，Go 模板页面仍应能继续使用。
